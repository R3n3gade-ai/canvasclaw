# 简介： **通用聊天界面，统一消息中枢**

JiuwenClaw 的​**频道**​，是您与不同聊天平台对话的**渠道**。JiuwenClaw 已经实现了**鸿蒙小艺、飞书**等channel的无缝接入，并持续拓展至更多元渠道。可以直接在通过 **飞书**、**鸿蒙生态终端的小艺app** 应用直接与JiuweClaw对话。

配置频道有两种方式：

* **网页配置**​（推荐）— 在 JiuwenClaw前端页面，点击 `智能体` / `频道`卡片，在频道管理模块，填写相关配置。

* **手动编辑 `config.yaml`** — 默认在 `~/.jiuwenclaw/config.yaml` （由首次 `jiuwenclaw-start` 启动时自动生成），将需要的频道设 `enabled: true` 并填好鉴权信息；保存后自动重载，无需重启。

## 9. 飞书多bot（`feishu_enterprise`）

当一个团队需要在同一套 JiuwenClaw 服务中接入多个飞书应用（多个 Bot）时，可使用 `channels.feishu_enterprise`。

每个 bot 会以独立 channel 运行，channel_id 格式为：`feishu_enterprise:<app_id>`。

当前多bot仅支持手动编辑 `~/.jiuwenclaw/config/config.yaml`：

``````
channels:
  feishu_enterprise:
    bot_a:
      app_id: "cli_xxx"
      app_secret: "xxx"
      encrypt_key: ""
      verification_token: ""
      allow_from: []
      enable_streaming: true
      chat_id: ""
      enabled: true
    bot_b:
      app_id: "cli_yyy"
      app_secret: "yyy"
      encrypt_key: ""
      verification_token: ""
      allow_from: []
      enable_streaming: true
      chat_id: ""
      enabled: true
``````

### 字段说明

| 配置项 | 说明 |
|:------|:-----|
| `app_id` | 飞书应用 App ID（必填） |
| `app_secret` | 飞书应用 App Secret（必填） |
| `encrypt_key` | 事件订阅加密 Key（可选） |
| `verification_token` | 事件订阅 Token（可选） |
| `allow_from` | 允许的用户 open_id 列表，空列表表示不限制 |
| `enable_streaming` | 是否启用流式下发 |
| `chat_id` | 固定推送目标（可选） |
| `enabled` | 是否启用该 bot |

### 与单 Bot `feishu` 的区别

- `feishu`：单通道，channel_id 固定为 `feishu`。  
- `feishu_enterprise`：多通道，每个 bot 使用独立 channel_id（`feishu_enterprise:<app_id>`）。  
- 多 Bot 场景下，会按 bot 维度分别记录最近会话信息，避免 bot 之间互相覆盖。

# Telegram

## 1. 创建 Telegram Bot

通过 Telegram 内置的 [@BotFather](https://t.me/BotFather) 创建机器人并获取 Bot Token。

第一步：在 Telegram 中搜索 `@BotFather` 并打开对话

![BotFather 搜索与对话界面](assets/images/TelegramBotFather.png)

第二步：发送 `/newbot` 命令，按提示输入机器人名称和用户名

BotFather 会依次要求你输入：
- **机器人显示名称**（如 `JiuwenClaw Bot`）
- **机器人用户名**（必须以 `bot` 结尾，如 `jiuwenclaw_bot`）

![BotFather 返回 Bot Token 的消息](assets/images/Telegram获取BotToken.png)

第三步：**保存 Bot Token**

创建成功后，BotFather 会返回一个格式如 `123456789:ABCDefGhIJKlmN...` 的 Token，请妥善保存，后续配置需要使用。

> ⚠️ **注意**：Bot Token 等同于机器人的密码，请勿泄露。如果 Token 泄露，可在 BotFather 中使用 `/revoke` 重新生成。

## 2. 绑定频道

**方式一：网页配置（推荐）**

在 JiuwenClaw 前端页面，点击 **`智能体` / `频道`** 卡片，在 Telegram 频道模块中填入 Bot Token，开启使用，点击保存即可。

![Telegram 频道配置界面](assets/images/Telegram频道配置.png)

**方式二：修改 `config.yaml`**

编辑 `~/.jiuwenclaw/config/config.yaml`：

``````
channels:
  telegram:
    # 从 @BotFather 获取的 Bot Token
    bot_token: "你的Bot Token"
    # 用户白名单（Telegram user_id），为空则允许所有用户
    allow_from: []
    # 消息解析模式: Markdown, HTML, None
    parse_mode: Markdown
    # 群聊模式: mention, reply, all, off
    group_chat_mode: mention
    enabled: true
``````

保存后若服务已运行会自动重载，未运行可以执行 `jiuwenclaw-start` 启动服务。

## 3. 配置说明

| 配置项 | 说明 | 默认值 |
|:------|:-----|:------|
| `bot_token` | 从 @BotFather 获取的 Bot Token，**必填** | 空 |
| `allow_from` | 允许使用的 Telegram user_id 白名单列表，为空则允许所有用户 | `[]` |
| `parse_mode` | 消息解析模式：`Markdown`、`HTML` 或 `None`（纯文本） | `Markdown` |
| `group_chat_mode` | 群聊响应模式（见下方详细说明） | `mention` |
| `enabled` | 是否启用 Telegram 频道 | `false` |

### 群聊模式说明

当机器人被添加到 Telegram 群组时，`group_chat_mode` 控制机器人如何响应群内消息：

| 模式 | 说明 |
|:-----|:-----|
| `mention` | **仅响应 @提及**：只有在群内 @机器人 时才会处理消息（推荐） |
| `reply` | **仅响应回复**：只有回复机器人消息时才会处理 |
| `all` | **响应所有消息**：群内所有文本消息都会被处理 |
| `off` | **禁用群聊**：在群组中不响应任何消息 |

## 4. 开始对话

方式1：在 Telegram 中搜索你创建的机器人用户名，发送 `你好` 即可开始对话

![Telegram 中与 Bot 对话界面](assets/images/Telegram对话界面.png)

方式2：将机器人添加到群组中，根据 `group_chat_mode` 配置与机器人交互

![Telegram 中与 Bot 对话群组界面](assets/images/Telegram群组对话界面.png)

## 5. 获取 user_id（配置白名单）

如需配置 `allow_from` 白名单，需要先获取用户的 Telegram user_id。可以通过以下方式获取：

1. 在 Telegram 中搜索 `@userinfobot`，发送任意消息即可获取你的 user_id
2. 将获取到的 user_id 填入 `allow_from` 列表中

``````
channels:
  telegram:
    bot_token: "你的Bot Token"
    allow_from:
      - "123456789"    # 用户A的 user_id
      - "987654321"    # 用户B的 user_id
    enabled: true
``````

> 💡 **提示**：`allow_from` 为空列表时，所有用户均可使用机器人。设置白名单后，只有列表中的用户才能与机器人对话。

# Discord

当前版本已经支持 Discord 频道接入。在「频道管理」中配置并启用 Discord Bot，也可以手动编辑 `config.yaml`。

**分步图文（开发者后台创建 Bot、Intent、安装链接、频道管理）见 [Discord.md](Discord.md)。**

- `bot_token`
- `application_id`
- `guild_id`
- `channel_id`
- `block_dm`
- `allow_from`
- `enabled`

可在 `~/.jiuwenclaw/config/config.yaml` 中按如下方式配置：

``````
channels:
  discord:
    bot_token: "Discord Bot Token"
    application_id: "Application ID"
    guild_id: "目标服务器 Guild ID"
    channel_id: "目标频道 Channel ID"
    block_dm: false
    allow_from: []
    enabled: true
``````

## 简略使用说明

1. 在 Discord Developer Portal 创建 Bot，获取 `bot_token`。
2. 在 Bot 页签开启 **Message Content Intent**。
3. 将机器人邀请进目标服务器并赋予读写频道权限。
4. 在 JiuwenClaw 里填入配置并启用 `enabled: true`。

## 字段说明

| 配置项 | 说明 |
|:------|:-----|
| `bot_token` | Discord Bot Token（必填） |
| `application_id` | 应用 ID（可选，建议填写） |
| `guild_id` | 仅监听指定服务器（可选，留空表示不限制） |
| `channel_id` | 仅监听/回复指定频道（可选，留空按消息上下文回复） |
| `block_dm` | 为 `true` 时不处理私信（DM） |
| `allow_from` | 允许访问的 Discord 用户 ID 列表，空列表表示允许所有 |
| `enabled` | 是否启用 Discord 频道 |

- 在安装 JiuwenClaw 的机器上的工作目录下新建空白文件夹，执行以下命令并等待完成
``````
mkdir wechat_channel
cd wechat_channel
npx -y @tencent-weixin/openclaw-weixin-cli@latest install
``````

![wechat_claw_install](assets/images/wechat_claw_install.png)


***注意***
- JiuwenClaw接入微信逻辑为底层协议转换，所以需要提前安装openClaw
- 执行命令成功后生成的二维码**不要扫码**

### 步骤 3：微信扫码接入

- 打开最新版本的 JiuwenClaw 前端点击「频道」->「微信」，启用微信配置功能并点击保存。页面会出现**二维码**。

![jiuwenclaw_enable_wechat](assets/images/jiuwenclaw_enable_wechat.png)

- 打开手机微信，依次进入「我」->「设置」->「插件」，在页面中找到 **微信ClawBot**，点击详情后可看到操作指导，其中第一步已经完成执行，直接点击 **开始扫一扫**，扫描上一步骤生成的二维码，继续连接即可完成接入！
 ![wechat_qrcode](assets/images/wechat_qrcode.png)
